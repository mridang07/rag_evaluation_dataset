# RAG Evaluation Dataset (Enterprise AI)

This repository contains a context-grounded RAG (Retrieval-Augmented Generation) dataset designed for evaluating LLM pipelines in enterprise AI systems.

## 🔹 Features
- Context-based QA pairs
- Covers document intelligence, vector search, agentic systems
- Includes evaluation metrics (accuracy, faithfulness, relevance)

---

## 📄 Dataset

### Context 1: Document Intelligence
Document intelligence systems process enterprise documents like invoices and audit reports using OCR and classification models.

**Q:** What is the role of OCR?  
**A:** OCR extracts text from scanned documents.

**Q:** What documents are processed?  
**A:** Invoices, audit reports, compliance filings.

---

### Context 2: RAG Systems
RAG combines retrieval with generation to improve accuracy.

**Q:** Why is RAG used?  
**A:** To improve accuracy using external data.

**Q:** Components of RAG?  
**A:** Vector DB, embeddings, LLM.

---

### Context 3: Vector Search
Vector DBs store embeddings for semantic search.

**Q:** Purpose of vector DB?  
**A:** Store embeddings for retrieval.

**Q:** Vector vs keyword search?  
**A:** Semantic vs exact match.

---

## 📊 Evaluation

| Metric | Description |
|--------|------------|
| Accuracy | Correctness of answer |
| Faithfulness | Grounded in context |
| Relevance | Matches query intent |

---

## 🔧 Methodology
This dataset uses context-grounded QA pairs to simulate real-world RAG pipelines. It is designed for evaluating LLM outputs in enterprise systems.
